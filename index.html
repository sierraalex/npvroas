Adstock (Exponential Decay)
Carryover effect of past spend
A(t) = S(t) + θ × A(t-1)

// Recursive: today's effective spend =
// today's actual spend + decayed yesterday
θ decay rate (0 = no memory, 1 = perfect memory)
A(t) adstocked spend at time t
Why it matters: TV and content have high θ (0.50-0.60) because brand impressions linger for weeks. Paid Search has low θ (0.20) because intent-based clicks have immediate, short-lived effects. Ignoring adstock misattributes long-memory channels' contributions.

Response Curves (Saturation)

More spend doesn't mean proportionally more customers. Diminishing returns are the law.

log
Log Response Curve
Most channels — concave saturation
R(x) = β × log(1 + k × x)

// Used for: Paid_Search, Paid_Social,
// Display, LinkedIn, Content, Organic
β scale (max response)
k sensitivity (how fast it saturates)
x adstocked spend
Why log: Simple, interpretable, and captures the core economics — doubling spend does NOT double output. The marginal customer always costs more than the average customer. This is why average ROAS misleads.
⇑
Hill Response Curve
Account-Based Marketing — S-curve with threshold
R(x) = β × xα / (xα + λ)

// S-shaped: no effect below threshold,
// then steep gains, then saturation
α steepness of S-curve
λ half-max spend (inflection point)
β max response ceiling
Why Hill for ABM: Enterprise ABM requires minimum investment (dedicated reps, custom content, event sponsorship) before seeing any pipeline. Below-threshold spend is wasted. The Hill function captures this activation energy requirement that log curves miss.
⚠

The Core Insight: Average Returns ≠ Marginal Returns

Account-Based Marketing shows 10.3x average LTV:CAC — the best of any channel. But its marginal NPV from +$50K additional spend is near zero because the response curve is saturated. LinkedIn at 8.0x average LTV:CAC delivers +$2.19M in marginal NPV per +$50K because it's still on the steep part of its curve. A CFO who allocates based on averages destroys value. A capital allocator who allocates based on marginals creates it.

Customer Lifetime Value (NPV)

The heart of NPVROAS: discounted, risk-adjusted cashflows over the customer lifetime

The NPVROAS Master Formula

NPVcustomer = Σt=035  [ (ACV/12) × (1 + g)t/12 × GM × Alive(t) ]  / (1 + r)t
ACV / 12
Annual contract value, monthly
(1 + g)t/12
Expansion revenue growth
GM
Gross margin (80% SaaS)
Alive(t)
Survival: retentiont (stochastic)
(1 + r)t
Discount factor at 10% annual
t = 0..35
36-month cashflow horizon
💰
Monthly Revenue per Customer
Base + expansion, pre-churn
Rev(t) = (ACV / 12) × (1 + g)t/12

// t = months since conversion
// g = annual expansion rate
// Enterprise: $12,500/mo growing 25%/yr
// Startup: $667/mo, no expansion
ACV annual contract value
g expansion rate (0-25%)
☠
Churn / Survival Function
Stochastic customer lifetime
P(alive at t) = Πi=0t Bernoulli(retention)

E[alive at t] = retentiont

// Enterprise 0.99^36 = 69.7% survive 3yr
// Startup 0.90^36 = 2.2% survive 3yr
Why stochastic, not deterministic: Using expected values (retentiont) gives you the mean. But capital allocation requires understanding the variance. Monte Carlo simulation of individual churn events reveals the risk profile of each channel's customer base. Enterprise is a bond; Startup is an option.
%
Discount Rate
Time value of money conversion
r_monthly = (1 + r_annual)1/12 - 1

// r_annual = 10% (hurdle rate / WACC)
// r_monthly = 0.797%

DF(t) = 1 / (1 + r_monthly)t
// Month 36 discount factor = 0.749
Why this matters: A dollar of revenue in month 36 is worth $0.75 today. ROAS treats $1 today and $1 in 3 years as equal. That's a 25% error on a single cashflow, compounded across thousands of customers. This is why ROAS systematically overvalues channels that acquire low-retention, slow-paying customers.

Cost & Efficiency Metrics

What you actually paid vs. what you actually got

📊
Customer Acquisition Cost (CAC)
True fully-loaded cost per customer
CAC(c) = Total_Spend(c) / Customers(c)

Fully_Loaded_CAC = CAC × Sales_Multiplier

// Sales multiplier by persona:
// Enterprise=1.5x, Mid-Market=1.3x
// SMB=1.1x, Startup=1.05x
Sales multiplier: Marketing spend alone underestimates true CAC. Enterprise deals require AEs, SEs, POC environments, legal review. The multiplier captures these costs. Without it, Enterprise channels look artificially efficient.
⚖
LTV:CAC Ratio
The classic — but incomplete — efficiency metric
LTV:CAC = NPV(Customer Cashflows) / CAC

// > 3.0x = healthy
// > 5.0x = excellent or under-investing
// < 1.0x = destroying value
The trap: LTV:CAC is an average metric. A channel at 10x LTV:CAC might be saturated (no room to grow) while a channel at 4x might have massive marginal upside. LTV:CAC tells you where you've been. Marginal NPV tells you where to go.

Marginal NPV & Capital Allocation

The decision that matters: where does the NEXT dollar go?

Δ
Marginal NPV per Channel
Monte Carlo incremental testing
ΔNPV(c) = E[NPV(budget + δ)] - E[NPV(budget)]

// Method: Monte Carlo (N simulations)
// 1. Run full pipeline at baseline budget
// 2. Add +$50K to channel c
// 3. Re-run full pipeline
// 4. Difference = marginal NPV
// 5. Average across N simulations
δ marginal spend increment ($50K)
N simulation count (3-25)
Why Monte Carlo: The response curve is nonlinear. The customer mix is stochastic. The churn paths are random. A closed-form marginal calculation would require assumptions we don't want to make. Simulation lets the model speak for itself.
⇄
Budget Reallocation
Moving capital from saturated to hungry channels
New_Budget(c) = Old_Budget(c) + Reallocation(c)

// Example scenario:
// Display: -60% ($180K freed)
// Account_Based: +70% of freed ($126K)
// LinkedIn_Ads: +30% of freed ($54K)

ΔPortfolio_NPV = NPV(new) - NPV(old)
// Result: +$984K incremental NPV

A/B Testing & Bayesian Updating

Continuous learning: updating beliefs about channel performance with new data

A|B
Frequentist A/B Test
Standard significance testing
z = (μB - μA) / √(sA²/nA + sB²/nB)

// H0: no difference between variants
// Reject if |z| > 1.96 (p < 0.05)

MDE = zα × √(2σ²/n)
// Minimum Detectable Effect at power 0.80
z test statistic
MDE smallest effect you can detect
n sample size per variant
𝒫
Bayesian Posterior Update
Updating channel beliefs with new evidence
P(θ|data) ∝ P(data|θ) × P(θ)

// Prior: Beta(α, β) on conversion rate
// Likelihood: Binomial(n, k)
// Posterior: Beta(α+k, β+n-k)

P(B > A) = ∫ P(θB > θA) dθ
// Probability variant B beats A
Why Bayesian for marketing: Frequentist testing answers "is there a difference?" Bayesian testing answers "how confident am I, and what should I do about it?" In capital allocation, the question is always the latter. A Bayesian framework lets you update channel performance beliefs continuously and make allocation decisions under quantified uncertainty — exactly like a portfolio manager.

NPVROAS(c) = Σcustomers from c  [ Σt=035 Rev(t) × GM × Alive(t) / (1+r)t ]  / Total_Spend(c)
